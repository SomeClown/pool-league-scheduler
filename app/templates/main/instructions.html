How to use Pool League Scheduler

Reading the Schedule

Schedule layout

Each season is organised into rounds — one per match night. Every round lists the matches for that night, grouped by bar. The date appears at the top of each round card.

Home and Away

Every match is played at the home team's bar. The home team is always listed first:

Home Team vs Away Team — Bar Name

The away team travels to the home team's bar for the match.

Byes

When there is an odd number of teams, one team sits out each round. That team's name appears in the Bye row at the bottom of the round card. Byes rotate so that every team sits out the same number of times over a full cycle.

Printing the schedule

Open any season and click Print Schedule at the top of the page. A print-formatted version of the full season opens automatically. On a desktop browser the print dialog appears immediately. On mobile, use your browser's share / print option.

Admin Guide

Setup order

Configure things in this order before creating a season:

  1. Bars — add each venue that hosts matches, and set its permanent table count.
  2. Teams — add each team and assign it a home bar.
  3. Season — once bars and teams exist, create a season and select which teams participate.
Season length

When creating a season you can define its length two ways:

  • End Date — pick a calendar date; the number of match nights is calculated automatically.
  • Number of Weeks — enter a count; the end date is calculated automatically.

Both fields stay in sync. The match nights preview below them shows how many rounds will be generated after excluding any blackout dates.

Per-season table caps

Each bar has a permanent table count set in the Admin panel. When creating a season you can optionally reduce the number of tables in use at any bar that has more than one table — for example, if only 2 of 3 tables are available for league play this season.

The cap only applies to the season being created; the bar's permanent count is not changed.

Blackout dates

Dates entered as blackouts are skipped entirely when the schedule is generated. Rounds are not lost — they shift forward so the full number of match nights is preserved.

Regenerate

The Regenerate button re-runs the scheduling algorithm with a fresh random seed. All season configuration — teams, dates, blackouts, and table caps — is preserved; only the match assignments change. Use this if you want a different arrangement of home/away assignments or round order.

Home / Away alternation across cycles

If a season runs long enough that the same two teams meet more than once, the algorithm flips their home/away assignment each time they meet. The team that was home in cycle 1 will be away in cycle 2, and so on.

Bar coverage goal

The scheduler tries to ensure that every bar has at least one of its home teams playing on each match night.

This goal has one unavoidable exception: if a bar has only one team and that team receives a bye in a given round, the bar will have no match that night. This is a mathematical consequence of odd team counts, not a scheduling defect.

Archiving seasons

Archiving a season removes it from the active list but preserves the full schedule permanently. Archived seasons can still be viewed and printed. They cannot be regenerated.
